# Grading Criteria

Use this markdown file to define how the LLM should grade each assignment.

## Instructions for the grader

1. **Read all files** in the student submission. The submission may include code, documents, presentations, or other materials.

2. **Evaluate** based on:
   - Correctness and completeness
   - Code quality (if applicable)
   - Clarity and organization
   - Adherence to requirements

3. **Provide**:
   - A clear grade (e.g., A, B, C, or a score out of 100)
   - Specific feedback on strengths and weaknesses
   - Constructive suggestions for improvement

## Example rubric (customize as needed)

| Criterion        | Weight | Description                          |
|------------------|--------|--------------------------------------|
| Correctness      | 40%    | Does the solution work correctly?    |
| Code/Doc Quality | 30%    | Is it well-written and organized?    |
| Completeness     | 20%    | Are all parts addressed?             |
| Creativity       | 10%    | Does it show understanding?          |
